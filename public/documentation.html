Basic Introduction of What the myFlix API does

It is the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Documentation
Description of what kind of data will be returned Endpoint URL HTTP Method Query Parameters Request body data format Response body data format
Get a list of movies /movies GET none none A JSON object holding data about all the movies
Get data about a single movie by title /movies/[title] GET /movies/:title none A JSON object holding data about a single movie, containing description, genre, director, image URL and whether it’s featured or not
Get data about a genre by title /movies/genre/[title] GET /movies/genre/:title none A JSON object holding data about a genre
Get data about a director by name /director/[name] GET /director/:name none A JSON object holding data about a director, containing bio, birth year and death year
Add data for a new user to register /users POST none A JSON object holding data about the user to add username, password, email and date of birth.

Add a movie to users list of favorite movies /users/[username]/movies/[moviesID] POST /users/:username/movies/:moviesID none A text message indicating that a movie was updated to the user favorite list
